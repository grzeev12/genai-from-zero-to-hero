## המסגרת

עד כה למדת לדבר עם AI. זה שינוי פרדיגמה: **AI שפועל לבד.**

**מה זה AI Agent:**
LLM רגיל: שאלה → תשובה. Agent: מטרה → תכנון → פעולות → בדיקה → תוצאה.

Agent יכול:
- לחפש מידע ב-web
- לקרוא ולכתוב קבצים
- לשלוח emails
- לקרוא ל-APIs
- לפעיל כלים (Jira, Slack, AWS, ServiceNow)
- להריץ קוד

**MCP: Model Context Protocol:**
הסטנדרט שקבע Anthropic (2024, נאמץ בתעשייה 2025) שמאפשר ל-AI לתקשר עם כלים ארגוניים. חשוב עליו כ-USB עבור AI: כל כלי שיש לו MCP connector: AI יכול להשתמש בו.

**Human-in-the-Loop (HITL): כלל שאי אפשר להתפשר עליו:**
- Low stakes + Reversible: Agent פועל לבד (סיכום, ניתוח, טיוטה)
- Medium stakes: Agent פועל + מודיע למנהל
- High stakes + Irreversible: Agent מציע, **מנהל מאשר** (מחיקת משאב, שליחת תקשורת לקוח, שינוי ייצור)

**Gartner 2026:** 40% מהאפליקציות הארגוניות יכילו AI Agents עד סוף 2026. זה לא עתיד: זה קורה עכשיו.

**שאלות לספק שמציע Agent Solution:**
1. מה ה-Agent יכול לעשות אוטונומית?
2. מה מחייב אישור אנושי?
3. איפה ה-audit trail?
4. מה קורה כשה-Agent טועה?
5. מה ה-cost model? (per-action? per-token?)
