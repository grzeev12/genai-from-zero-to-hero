## המסגרת

L2 מסתיים בשאלה שכל מנהל חייב לדעת לענות: "כמה ה-AI עולה לנו ומה אנחנו מקבלים בתמורה?" זו לא שאלה טכנית: זו שאלה ניהולית.

**עלויות ישירות:**

| כלי | עלות חודשית | Per user |
|-----|------------|---------|
| Claude Pro | $20 ($17 בתשלום שנתי) | Per user |
| ChatGPT Plus | $20 | Per user |
| Google AI Pro (Gemini) | $19.99 | Per user |
| Microsoft Copilot M365 | $30 | Per user |
| Claude API (pay-per-use) | תלוי מודל, פער של עד פי 2.5 בין הספקים | Per usage |

**עלויות נסתרות:**
- זמן הטמעה ואימון (~20 שעות per user בהתחלה)
- עלות Integration אם בונים מעל API
- עלות Data Preparation (להכין context, מסמכים)
- עלות Governance (לכתוב policy, לנהל)

**ROI Calculation Framework:**
> שעות שנחסכות בשבוע x עלות שעה x 52 = ערך שנתי
> ערך שנתי / עלות שנתית = ROI

**דוגמה:** מנהל חוסך 5 שעות/שבוע. עלות שעה (loaded): ₪150. 
5 x 150 x 52 = ₪39,000 ערך. עלות Claude Pro: ₪900/שנה. ROI: 43x.

**בנצ'מרק עדכני (2026):** מחקר על עובדי ידע שמשתמשים ב-production AI agents (עם telemetry אמיתי, לא סקרים) מוצא חיסכון חציוני של 6.4 שעות בשבוע לכל seat. במקביל, ארגונים מאבדים בממוצע 51 ימי עבודה לעובד בשנה בגלל "technology friction", חוסר יעילות בשילוב בין כלים. הפער הזה הוא בדיוק המקום שבו ROI Analysis טוב עוזר: לא רק למדוד כמה AI חוסך, אלא גם כמה זמן הולך לאיבוד בגלל אי-שילוב נכון.

**Build vs Subscribe Decision:**
- Subscribe (SaaS): מתחת ל-100 users, use case ברור, אין data privacy issue
- Build on API: מעל 100 users, use case ייחודי, נתונים רגישים, רוצים שליטה

**Prompt לניתוח ROI:**
> "אני מנהל X אנשים. כל אחד חוסך Y שעות בשבוע עם AI. עלות שעה Z. עלות הכלים: [פרט]. חשב ROI שנתי ל-3 תרחישים: adoption של 25%/50%/100% בצוות."

## מה לעשות

בנה ROI Analysis לצוות שלך:

**שלב 1:** רשום את כל הכלים הנוכחיים + עלותם.
**שלב 2:** בקש מ-2-3 אנשים בצוות להעריך כמה שעות הם חוסכים בשבוע.
**שלב 3:** חשב ROI עם ה-framework + AI לבדיקת המספרים.
**שלב 4:** זהה use case אחד שה-ROI שלו נמוך: מדוע? מה לשנות?
**שלב 5:** בנה Business Case לתוספת כלי או הרחבת רישוי. הצג לCFO.
