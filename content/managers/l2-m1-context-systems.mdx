## המסגרת

בכל שיחה חדשה עם AI, אתה מסביר מחדש מי אתה, מה הארגון, מה הסגנון שלך. זה בזבוז. Context Systems פותר את זה.

**Claude Projects (הכלי הרלוונטי):**
שיחה + memory + הוראות קבועות שנשמרות. כל שיחה חדשה ב-Project יורשת את ה-System Prompt ואת המסמכים שהעלית.

**מה לשים ב-System Prompt:**

**זהות ותפקיד:**
"אני [שם], מנהל ענן ב-[חברה]. האחריות שלי: [תחומים]. הצוות שלי: [X אנשים, התמחויות]."

**הקשר ארגוני:**
"הארגון: [גודל, תעשייה, ספקי ענן]. המחסנית הטכנית: [AWS/Azure/GCP, כלים עיקריים]. הספקים העיקריים שאנחנו עובדים איתם: [רשימה]."

**סגנון (מ-L1-M7):**
"סגנון התקשורת שלי: [ישיר/רשמי/מקצועי]. כתוב בעברית עם מינוח טכני באנגלית. לא להתחיל ב'בהחלט' או 'כמובן'. משפטים קצרים."

**הנחיות קבועות:**
"תמיד: (1) ציין כשאתה לא בטוח, (2) הצע שאלות follow-up, (3) אל תמציא מספרים ספציפיים."

**Projects מרכזיים לבנות:**
1. **Project ראשי** - עבודה יומיומית כללית
2. **Project FinOps** - כל עבודת עלויות ענן (הוסף Cost Reports כמסמכים)
3. **Project ספקים** - ניהול SOW, חוזים, תקשורת עם ספקים

## מה לעשות

בנה שני Projects השבוע:

**Project 1 - ראשי:**
כתוב System Prompt שכולל: זהות, הקשר ארגוני, style guide (מ-L1-M7), הנחיות קבועות.
Test: שאל 5 שאלות שגרתיות. האם הפלט כבר מרגיש "שלך"?

**Project 2 - מיוחד (לבחור):**
בחר domain אחד: FinOps / ניהול ספקים / תקשורת C-suite.
בנה System Prompt ספציפי + העלה מסמך רלוונטי אחד.
