## כלים לחקור

> **תרגיל:** בנה Auto-Remediation Playbook לתרחיש ספציפי.

**AIOps Setup:**
- **Claude** - "אתה SRE ב-enterprise. עזור לי לבנות AIOps Architecture ל-EKS Cluster: מה כלי ה-monitoring? כיצד מוגדר Anomaly Detection? מה ה-Auto-Remediation Playbooks לאירועים הנפוצים?"

**Auto-Remediation Playbook:**
```yaml
# דוגמה: Event-Driven Ansible Rule
- name: OOMKilled Pod Remediation
  condition:
    event:
      alert.name: "pod_oomkilled"
  action:
    run_job_template:
      name: "Increase Memory Limit"
      extra_vars:
        pod: "{{ event.pod_name }}"
        namespace: "{{ event.namespace }}"
```
- **Claude** - "כתוב Auto-Remediation Playbook ל: (1) Pod OOMKilled - גדל Memory, (2) High CPU - Scale Out, (3) Disk > 80% - Alert escalation. לכל אחד: Trigger, Action, Rollback, Escalation condition."

## מטלת הגשה

**AIOps Design. עמוד וחצי.**

1. **Architecture:** תאר את ה-AIOps Architecture שאתה מציע לסביבה שלך
2. **Playbooks:** שלושה Auto-Remediation Playbooks עם Trigger, Action, Rollback
3. **Autonomy Boundaries:** מה AI מחליט לבד? מה הוא מסלים לאנשים? למה?
4. **ראיון שאלות:** מה שאלת כשבנית זאת? מה ה-AI ענה שהפתיע אותך?
