## מה תלמד

בסוף המודול הזה אתה מסוגל:
- לשלב AI בתוך CI/CD Pipeline בפועל
- להשתמש ב-AI ל-Test Generation ו-Security Scanning בתוך Pipeline
- למדוד את ההשפעה של שינויים ב-Pipeline

---

## מקורות לימוד

> **מגמת 2025:** CI/CD הפך ל-AI-CI/CD. Pipeline שלא כולל AI-powered test generation וsecurity scanning הוא Pipeline שמאחר.

**על AI ב-CI/CD:**
- [GitHub Actions - AI Integration](https://docs.github.com/en/actions) - כיצד לשלב AI steps ב-GitHub Actions
- [GitLab CI/CD - AI Features](https://docs.gitlab.com/ee/ci/) - AI Code Review ב-Merge Request
- חפש: `"AI pipeline optimization 2025"` - כלים ופרקטיקות
- [Snyk - AI Security in CI/CD](https://snyk.io/) - סריקת אבטחה עם AI בתוך Pipeline

**AI Test Generation:**
- [Diffblue Cover](https://www.diffblue.com/) - AI Test Generation ל-Java
- [Copilot - Test Generation](https://docs.github.com/en/copilot) - כיצד Copilot כותב Tests
- חפש: `"AI test generation CI pipeline 2025"` - שימוש ב-AI ל-tests ב-pipeline

---

## כלים לחקור

> **תרגיל:** הוסף שלב אחד ל-Pipeline שמשתמש ב-AI.

**AI ב-GitHub Actions:**
```yaml
# דוגמה: AI Code Review ב-GitHub Actions
- name: AI Code Review
  uses: github/copilot-pull-request-summary@v1
```

- **Claude / ChatGPT** - "הנה ה-GitHub Actions Pipeline שלי: [הדבק]. מה אפשר לשפר? מה Bottlenecks אפשריים? כיצד להוסיף AI-powered security scanning?"
- **Snyk** - חבר Snyk ל-GitHub repo שלך. הפעל scan ב-Pull Request.
- **CodeClimate / SonarQube** - בדוק אם יש AI features בכלים שאתה כבר משתמש

---

## מטלת הגשה

**Pipeline שיפור. עמוד אחד.**

1. **Pipeline מקורי:** תאר את ה-Pipeline שלך - Steps, זמן ריצה, כישלונות נפוצים
2. **שיפור עם AI:** מה שינית? מה הוספת? כיצד AI עזר בעיצוב השינוי?
3. **שלב AI חדש:** מה ה-AI-powered step שהוספת? מה הוא עושה?
4. **מדידה:** לפני ואחרי - זמן, כישלונות, Coverage
