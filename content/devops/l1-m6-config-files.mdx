## המסגרת

קובצי קונפיגורציה (YAML, JSON, TOML, XML) הם השפה שבה תשתיות "מדברות". טעות קטנה, indentation שגוי, שדה חסר, גורמת לכשל שקשה לאתר.

**מבנה בסיסי:**
- YAML: רגיש ל-indentation, נפוץ ב-Kubernetes, Docker Compose, GitHub Actions
- JSON: מובנה, נפוץ ב-APIs וקונפיגורציית אפליקציות
- TOML: קריא יותר לבני אדם, נפוץ בכלי Rust/Python
- XML: מבנה מורכב יותר, עדיין נפוץ במערכות legacy

**Prompt לניתוח קובץ קונפיגורציה:**
> "הנה קובץ [Dockerfile / values.yaml / config.json]. הסבר מה כל רכיב עושה. זהה: (1) שגיאות syntax, (2) הגדרות חסרות שבדרך כלל נדרשות, (3) best practices שלא מיושמות."

**יצירת תבניות לשימוש חוזר:** אחרי שהבנת מבנה קובץ מסוים, בקש מ-AI ליצור תבנית (template) גנרית שניתן למלא לכל פרויקט חדש, במקום להתחיל מאפס בכל פעם.

## מה לעשות

בחר קובץ קונפיגורציה מורכב מהעבודה שלך (Dockerfile, values.yaml, או קובץ הגדרות אחר עם 50+ שורות).

**תרגיל:** העלה ל-AI ובקש הסבר מלא, רכיב אחר רכיב. שאל: "מה חסר? מה לא best practice?"

בנה תבנית קונפיגורציה גנרית (template) לשימוש חוזר, בליווי AI, לסוג הקובץ הזה.
