## המסגרת

פייפליינים מורכבים דורשים YAML נכון, שילוב כלי בדיקה, וטיפול נכון ב-Secrets. AI מאיץ משמעותית את הכתיבה והדיבוג.

**בניית פייפליין:**
> "אתה DevOps Engineer. כתוב פייפליין GitHub Actions ל-[תיאור]. כלול: build, test, security scan (Trivy), deploy ל-[סביבה], עם approval gate לפני production."

**כלי בדיקה משולבים:**
- Sonarqube: איכות קוד וסריקת bugs
- Trivy: סריקת חולשות באימג'ים ובתלויות

**ניתוח כשלי ריצה:** הדבק את ה-log המלא של הריצה שנכשלה (לא רק שורת השגיאה האחרונה), בקש מ-AI לזהות את השלב שנכשל ולמה.

**אסטרטגיות פריסה:**
- Blue-Green: שתי סביבות מלאות, מעבר מיידי
- Canary: פריסה הדרגתית לאחוז קטן מהתעבורה

**Secrets בפייפליין:** לעולם לא hardcode. השתמש ב-Secret managers (GitHub Secrets, Vault, AWS Secrets Manager). AI עוזר לתכנן את הזרימה אבל אל תיתן לו לראות את הערכים עצמם.

## מה לעשות

**שלב 1:** בנה קובץ YAML מלא לפייפליין GitHub Actions שמבצע build, test, security scan, ו-deploy לענן.

**שלב 2:** אם יש לך פייפליין שבור, נתח אותו עם AI. אם לא, צור תרחיש שגיאה מדומה ותקן.

**שלב 3:** תכנן אסטרטגיית Blue-Green או Canary לשירות אחד. תעד את ההחלטה.
