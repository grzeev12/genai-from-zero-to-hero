## המסגרת

כשמשימה מורכבת מדי לסוכן בודד, פותרים עם Multi-Agent: כמה סוכנים מתמחים שמתקשרים ביניהם.

**פרימוורקים מובילים:**
- CrewAI: הגדרת Crew של agents עם תפקידים ברורים
- LangGraph: אורקסטרציה מבוססת גרף למשימות מורכבות עם ענפים לוגיים

**חלוקת Personas לתשתיות:**
- Agent ארכיטקטורה: מעצב את הפתרון הטכני
- Agent SecOps: בודק כל הצעה מול מדיניות אבטחה
- Agent FinOps: מעריך עלות לפני אישור

**פרוטוקולי תקשורת:** כל agent מקבל תפקיד, יעד, וכלים ספציפיים. יש לתכנן כיצד הם מעבירים מידע (structured output, shared memory) ואיך מתקבלת החלטה סופית כשיש מחלוקת.

**סיכוני Multi-Agent (מ-L3-M4, בקנה מידה גדול יותר):**
- Cascading Failures: שגיאה בסוכן אחד מתפשטת
- Accountability Gap: קשה לדעת איזה סוכן אחראי להחלטה
- Cost Explosion: כל sub-agent מוסיף עלות tokens

**אינטגרציה עם כלי צוות:** Slack להתראות, Jira לפתיחת tickets אוטומטית, כדי שהמערכת תשתלב בזרימת העבודה הקיימת ולא תרוץ בבידוד.

## מה לעשות

**שלב 1:** תכנן ועצב מערכת מרובת סוכנים (3+ סוכנים) שמבצעת אודיט תשתיתי מלא לענן.

**שלב 2:** בנה מנגנון ניטור על תהליך התקשורת בין הסוכנים.

**שלב 3:** תעד תרחיש כשל אחד: מה קורה כשסוכן אחד טועה, ואיך המערכת מגיבה.
