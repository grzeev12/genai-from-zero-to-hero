## מה תלמד

בסוף המודול הזה אתה מסוגל:
- לכתוב Terraform Modules מלאים עם AI כעוזר
- לשלב AI-powered Security Scanning על IaC
- לייצר תיעוד IaC אוטומטי עם AI

---

## מקורות לימוד

> **המגמה:** AI Copilots שינו את כתיבת IaC - מנוסים כותבים Module בעשר דקות במקום שעה. הבנת הקוד עדיין נדרשת.

**על AI ו-IaC:**
- [HashiCorp - AI for Terraform](https://developer.hashicorp.com/terraform/docs) - חפש AI features
- [Checkov](https://www.checkov.io/) - Static Analysis ל-IaC
- [tfsec](https://aquasecurity.github.io/tfsec/) - Security Scanning ל-Terraform
- חפש: `"AI terraform module generation 2025"` - best practices
- [terraform-docs](https://terraform-docs.io/) - תיעוד אוטומטי ל-Terraform

**על IaC Security:**
- חפש: `"IaC security scanning AI integration 2025"` - כלים ו-practices
- [AWS Config Rules](https://aws.amazon.com/config/) - Compliance checks על IaC

## כלים לחקור

> **תרגיל:** כתוב Terraform Module שמייצג תשתית שכבר קיימת בארגון שלך.

**כתיבת Module:**
- **Claude** - "כתוב Terraform Module לDeployment של [תאר: RDS PostgreSQL / EKS Node Group / VPC עם subnets]. כלול: variables עם descriptions ו-types, locals, resources, outputs. כלול tagging strategy. Follow AWS Well-Architected Framework."
- **GitHub Copilot** - פתח קובץ `main.tf` ריק. כתוב comment המתאר את ה-module. ראה מה מוצע.

**Security:**
```bash
# הרץ Checkov על הקוד שיצרת:
checkov -d . --framework terraform
```
- **Claude** - "הנה ממצאי Checkov: [הדבק]. תסביר כל ממצא ואיך לתקן."

**תיעוד:**
- **terraform-docs** - `terraform-docs markdown . > README.md`
- **Claude** - "כתוב README ל-Terraform Module הזה בפורמט standard: description, usage example, inputs table, outputs table."

## מטלת הגשה

**Module מלא. עמוד וחצי.**

1. **Module:** הצג את ה-module שכתבת (תיאור + structure - לא כל הקוד). מה AI כתב? מה שינית?
2. **Security Scan:** מה Checkov/tfsec מצא? מה תיקנת? מה בחרת לא לתקן ולמה?
3. **תיעוד:** הצג את ה-README שנוצר. האם AI עזר? מה שיפרת?
4. **לקח:** מה תעשה אחרת ב-Module הבא?
