## מה תלמד

בסוף המודול הזה אתה מסוגל:
- לבצע AI-assisted Threat Modeling על architectures
- לבנות Security Gate ב-CI/CD שעוצר vulnerabilities לפני Production
- לנהל False Positives בכלי Security Scanning

## כלים לחקור

> **תרגיל:** בחר Architecture שאתה מכיר. בצע Threat Model עם AI.

**Threat Modeling:**
- **Claude** - "אתה Security Architect. בצע STRIDE Threat Model על ה-Architecture הזה: [תאר: microservices ב-EKS, API Gateway, RDS, S3]. לכל component: מה האיום? מה ה-Mitigation?"

**Security Gate ב-Pipeline:**
```yaml
# GitHub Actions - Security Stage
security-scan:
  runs-on: ubuntu-latest
  steps:
    - uses: actions/checkout@v4
    - name: Run Trivy vulnerability scanner
      uses: aquasecurity/trivy-action@master
      with:
        image-ref: ${{ env.IMAGE }}
        exit-code: '1'
        severity: 'CRITICAL,HIGH'
```
- **Claude** - "עזור לי לבנות Security Pipeline Stage שכולל: Container scanning, IaC scanning, SAST. מה הכלים הנכונים? מה ה-thresholds המומלצים?"

**False Positives:**
- **Claude** - "הנה ממצאי Trivy: [הדבק]. אילו מהם הם False Positives? כיצד מסמנים אותם ב-`.trivyignore`?"

## מטלת הגשה

**DevSecOps Implementation. עמוד וחצי.**

1. **Threat Model:** בחר system אחד. הצג 3-5 איומים שה-AI זיהה. מה היה מפתיע?
2. **Security Gate:** מה הוספת ל-Pipeline? מה הגדרות ה-threshold? מה היה הממצא הראשון?
3. **False Positives:** כמה False Positives קיבלת? כיצד טיפלת בהם? מה הכלל שקבעת?
4. **ROI:** כמה זמן לקח להוסיף את ה-Gate? האם זה כדאי?
