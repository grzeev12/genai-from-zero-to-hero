## המסגרת

Agentic DevOps הוא השינוי הגדול ביותר בתחום כרגע. Agent לא רק עונה, הוא מתכנן, מפעיל כלים, ובודק תוצאה.

**Tool Calling:** מודל שפה שמקבל גישה לכלים חיצוניים (AWS CLI, kubectl, Terraform) ומחליט בעצמו מתי ואיך להפעיל אותם, על בסיס מטרה שהוגדרה לו.

**Protocols רלוונטיים:**
- **MCP (Model Context Protocol)**: מאפשר ל-AI לתקשר עם כלים ארגוניים
- **A2A (Agent2Agent Protocol)**: הסטנדרט שגוגל הרחיבה ב-Google Cloud Next 2026, לתקשורת בין agents שונים

**המספר שאסור להתעלם ממנו:** 74% מפריסות ה-Agent בארגונים נסוגות (rolled back) בגלל בעיות ממשל וסיכונים, לפי ניתוח GSPANN 2026. Agentic DevOps בלי guardrails הוא מתכון לאסון.

**סיכוני Privilege Escalation:** Agent שמקבל גישה רחבה מדי ל-CLI עלול לבצע פעולות שלא התכוונת אליהן. תמיד least privilege: מה בדיוק ה-agent יכול לעשות, לא יותר.

**Guardrails נגד לולאות אינסופיות:** הגדר max iterations, timeout, ו-circuit breaker שעוצר את הסוכן אם הוא "תקוע" בניסיון חוזר.

**Authority Boundary (מ-MCM Framework):**
- אוטונומי לגמרי: קריאה בלבד, ניתוח, דוחות
- מודיע אחרי פעולה: פתיחת ticket, סיכום
- מציע וממתין לאישור: כל דבר שמשנה production
