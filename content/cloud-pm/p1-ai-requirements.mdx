## מה תלמד

בסוף המודול הזה אתה מסוגל:
- לכתוב User Stories איכותיות יותר עם עזרת AI
- לייצר Acceptance Criteria ברורים ומדידים
- להשתמש ב-AI לזיהוי פערים ב-Requirements

---

## מקורות לימוד

> **הבעיה הנפוצה ב-Cloud Projects:** Requirements גנריים שמגיעים מה-business ו-Developers צריכים לפרש. AI יכול לגשר.

**על User Stories ו-AI:**
- חפש: `"AI user story generation acceptance criteria 2025"` - best practices
- [Atlassian - User Stories Guide](https://www.atlassian.com/agile/project-management/user-stories) - הבסיס
- חפש: `"INVEST criteria user stories AI 2025"` - כיצד AI מוודא INVEST compliance
- חפש: `"AI requirements analysis cloud migration 2025"` - Requirements ל-cloud projects

---

## כלים לחקור

> **תרגיל:** קח Requirement מהפרויקט שלך. הפוך אותו ל-User Story עם AI.

**כתיבת User Stories:**
- **Claude** - "הנה Requirement: [הדבק]. הפוך אותו ל-User Story בפורמט 'As a [user], I want [action], so that [value]'. וודא INVEST compliance. הוסף Acceptance Criteria בפורמט Gherkin."
- **ChatGPT** - "הנה Epic של פרויקט [תאר]. פרק ל-5 User Stories. לכל אחת: Story, Acceptance Criteria, Definition of Done, Story Points הערכה."

**Gap Analysis:**
- **Claude** - "הנה Requirements Document של פרויקט [תאר]: [הדבק]. מה חסר? מה לא ברור? מה יכול לייצר מחלוקות בין הצוות?"

---

## מטלת הגשה

**User Stories + Gap Analysis. חצי עמוד.**

1. **3 User Stories:** שכתבת עם AI לפרויקט שלך. לכל אחת: Story + 3 Acceptance Criteria
2. **Gap Analysis:** מה ה-AI מצא ב-Requirements הקיימים שלך?
3. **שיפור:** מה שינית בגרסה הסופית ביחס למה ש-AI הציע?
